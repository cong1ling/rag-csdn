# RAG 升级优化文档

更新时间：2026-04-10

## 目标

针对当前 `rag-bilibili` 的检索增强问答链路，优先落地低成本高回报优化，并持续在本文档中同步状态。

## 本轮已完成

### 1. 检索阈值配置化

- 使用 `spring.ai.alibaba.dashvector.similarity-threshold` 作为聊天检索阈值来源
- 默认值调整为 `0.35`
- 聊天检索链路不再写死 `0.0`

### 2. ba Rewrite

- 新增 `rag.chat.query-rewrite-enabled` 配置项
- 当存在历史消息时，先让模型将当前问题改写为独立检索查询
- 改写失败时回退到用户原始问题，不阻断主流程

### 3. 上下文元数据增强

- 导入向量时补充 `title`、`totalChunks` 元数据
- 回答上下文中展示：
  - 视频标题
  - BV 号
  - 片段序号
  - 相似度
- 说明：历史上已经导入的向量数据若缺少新增元数据，需重新导入后才能完整展示 `totalChunks`

### 4. Prompt 强化

- 要求基于检索片段回答
- 要求关键结论附带来源
- 若多个片段冲突，单独指出不一致
- 若信息不足，明确说明无法完整回答并仅给出已知部分

## 待办建议

### 5. Chunk overlap

- 推荐改为支持 overlap 的切分方案
- 推荐参数：`chunk_size=512`，`overlap=128`
- 当前进展：新导入链路已支持配置化切分参数，默认 `chunk-size=512`、`overlap-chars=128`
- 已补充历史数据重建入口：对已有视频可重新抓取字幕、重新切分并重写向量索引，使旧数据同步应用最新 overlap 策略
- 使用方式：在视频列表中对目标视频执行“重建索引”，并重新提供 B 站访问凭证

### 6. Hybrid Search

- 增加关键词检索通道
- 与向量召回结果做归并、去重和排序
- 优先解决术语、专有名词、数字型问题的召回不足
- 当前进展：已新增可配置 `hybrid-search-enabled`、`keyword-top-k`，在聊天检索链路中默认启用混合检索

### 7. Rerank

- 候选召回 `Top-20`
- 经 rerank 后保留 `Top-5`
- 优先用于多视频场景和复杂分析型问题
- 当前进展：已接入可配置规则型 rerank，基于标题命中、正文命中、关键词覆盖率和初始召回分数做二阶段重排
- 已补充模型式 rerank 开关：在规则重排结果上可继续对前 `N` 个候选做模型式精排，默认关闭，按需开启
- 推荐配置：
  - `rag.chat.model-rerank-enabled=false`
  - `rag.chat.model-rerank-top-k=8`

### 8. 动态 Top-K

- 简单事实问题：`Top-3`
- 常规说明问题：`Top-5`
- 综合分析问题：`Top-8`
- 当前进展：已在聊天检索链路中默认启用，结合问题复杂度和 `ALL_VIDEOS` 会话做动态调参

## 验证建议

- 准备一组单轮问答样例，观察阈值上调后是否减少无关片段
- 准备一组多轮省略指代样例，验证 Query Rewrite 的召回改善
- 检查回答是否稳定带来源引用，且能在信息不足时明确说明边界
