# 前端快速启动文档

## 1. 适用范围

本文档用于快速启动 `rag-bilibili-front` 前端项目，适合本地开发、自测联调和构建验收。

前端目录：

```text
rag-bilibili-front/
```

技术栈：

- Vue 3
- Vite
- Vue Router
- Pinia
- Element Plus
- Axios

## 2. 环境要求

建议准备以下环境：

- Node.js 18 及以上
- npm 9 及以上

查看版本：

```bash
node -v
npm -v
```

## 3. 安装依赖

在项目根目录执行：

```bash
cd rag-bilibili-front
npm install
```

## 4. 配置环境变量

前端示例环境文件：

```text
rag-bilibili-front/.env.example
```

当前示例配置：

```env
VITE_API_BASE_URL=/api
VITE_PROXY_TARGET=http://localhost:8080
```

建议复制为本地开发文件：

```bash
cd rag-bilibili-front
copy .env.example .env
```

参数说明：

- `VITE_API_BASE_URL`：前端请求接口的基础路径，默认使用 `/api`
- `VITE_PROXY_TARGET`：Vite 开发服务器代理目标，默认转发到 `http://localhost:8080`

## 5. 启动开发环境

先启动后端服务，再启动前端：

```bash
cd rag-bilibili-front
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

当前 `vite.config.js` 中的开发服务器配置为：

- 监听地址：`0.0.0.0`
- 端口：`5173`
- 代理前缀：`/api`

## 6. 与后端联调

默认联调方式如下：

1. 浏览器访问前端 `http://localhost:5173`
2. 前端将 `/api/*` 请求代理到 `http://localhost:8080`
3. 登录态通过 Cookie 传递，Axios 已开启 `withCredentials: true`
4. 流式问答接口通过 `fetch + SSE` 方式请求同一前缀下的接口

如果后端端口不是 `8080`，只需要修改：

```env
VITE_PROXY_TARGET=http://你的后端地址
```

## 7. 可用页面

启动成功后，可访问以下主要页面：

- `/login`：登录
- `/register`：注册
- `/import`：导入视频
- `/videos`：视频列表
- `/sessions`：会话列表
- `/chat/:sessionId`：流式问答页

未登录访问业务页面时，会自动跳转到登录页。

## 8. 构建生产包

执行：

```bash
cd rag-bilibili-front
npm run build
```

构建产物默认输出到：

```text
rag-bilibili-front/dist/
```

## 9. 常见问题

### 9.1 前端启动了但接口报错

优先检查：

- 后端是否已启动
- `VITE_PROXY_TARGET` 是否正确
- 后端是否允许当前来源的跨域/凭证访问
- 浏览器是否携带了登录 Cookie

### 9.2 登录后又跳回登录页

当前前端依赖服务端登录态校验，需确认：

- 后端 `/api/auth/current` 能正常返回当前用户
- Cookie 没有被浏览器拦截
- 前后端域名、端口、协议是否与后端会话配置匹配

### 9.3 聊天页面没有流式输出

优先检查：

- 后端流式接口是否可用
- 浏览器控制台是否出现网络错误
- 代理后的接口路径是否正确
- 返回内容是否符合 SSE 事件流格式

## 10. 推荐启动顺序

```bash
cd rag-bilibili-server
# 启动后端

cd ../rag-bilibili-front
copy .env.example .env
npm install
npm run dev
```
