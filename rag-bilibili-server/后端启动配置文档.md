# RAG-Bilibili 后端启动配置文档

## 前置要求

- Java 17+
- MySQL 8.0+
- Maven 3.6+

## 配置步骤

### 1. 数据库配置

修改 `src/main/resources/application.yml` 中的数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/rag_bilibili?createDatabaseIfNotExist=true&useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: your_username  # 修改为你的数据库用户名
    password: your_password  # 修改为你的数据库密码
```

**注意**：数据库会自动创建（createDatabaseIfNotExist=true），表结构由 Flyway 自动迁移。

### 2. OpenAI 兼容模型配置

项目支持任何 OpenAI 格式的 API 接口。配置环境变量：

```bash
# Windows (CMD)
set OPENAI_API_KEY=your_api_key
set OPENAI_BASE_URL=https://your-api-endpoint.com/v1

# Windows (PowerShell)
$env:OPENAI_API_KEY="your_api_key"
$env:OPENAI_BASE_URL="https://your-api-endpoint.com/v1"

# Linux/Mac
export OPENAI_API_KEY=your_api_key
export OPENAI_BASE_URL=https://your-api-endpoint.com/v1
```

**推荐服务商**：
- DeepSeek API（默认模型：deepseek-v3.2）
- OpenAI 官方
- 其他兼容 OpenAI 格式的服务

如需更换模型，修改 `application.yml`：

```yaml
spring:
  ai:
    openai:
      chat:
        options:
          model: your-model-name  # 修改为你想使用的模型
```

### 3. 阿里云 DashVector 配置（向量数据库）

#### 3.1 开通 DashVector 服务

1. 登录 [阿里云控制台](https://www.aliyun.com/)
2. 搜索并进入 **DashVector** 服务
3. 开通服务（可免费试用）
4. 创建 **Collection**（集合）：
   - Collection 名称：`bilibili`（或自定义，需同步修改 yml）
   - 向量维度：`1024`（与 text-embedding-v4 模型匹配）
   - 距离度量：推荐 `cosine`

#### 3.2 获取 API Key 和 Endpoint

在 DashVector 控制台：
- 复制 **API Key**
- 复制 **Endpoint**（格式如：`vrs-cn-xxxxx.dashvector.cn-hangzhou.aliyuncs.com`）

#### 3.3 配置环境变量

```bash
# Windows (CMD)
set DASHVECTOR_API_KEY=your_dashvector_api_key
set DASHVECTOR_ENDPOINT=your_dashvector_endpoint

# Windows (PowerShell)
$env:DASHVECTOR_API_KEY="your_dashvector_api_key"
$env:DASHVECTOR_ENDPOINT="your_dashvector_endpoint"

# Linux/Mac
export DASHVECTOR_API_KEY=your_dashvector_api_key
export DASHVECTOR_ENDPOINT=your_dashvector_endpoint
```

如需修改 Collection 名称，编辑 `application.yml`：

```yaml
spring:
  ai:
    alibaba:
      dashvector:
        collection: your_collection_name  # 修改为你的 Collection 名称
```

### 4. 阿里云百练 DashScope 配置（向量模型）

#### 4.1 获取 API Key

1. 登录 [阿里云百练控制台](https://dashscope.console.aliyun.com/)
2. 进入 **API-KEY 管理**
3. 创建或复制已有的 API Key

#### 4.2 配置环境变量

```bash
# Windows (CMD)
set DASHSCOPE_API_KEY=your_dashscope_api_key

# Windows (PowerShell)
$env:DASHSCOPE_API_KEY="your_dashscope_api_key"

# Linux/Mac
export DASHSCOPE_API_KEY=your_dashscope_api_key
```

#### 4.3 向量模型说明

默认使用 **text-embedding-v4** 模型（1024 维）。如需更换模型，修改 `application.yml`：

```yaml
spring:
  ai:
    dashscope:
      embedding:
        options:
          model: text-embedding-v3  # 可选：v1/v2/v3/v4
          dimensions: 1024  # 根据模型调整维度
```

**注意**：更换模型后需确保 DashVector Collection 的维度与模型匹配。

## 启动项目

### 方式一：Maven 启动

```bash
cd rag-bilibili-server
mvn spring-boot:run
```

### 方式二：IDEA 启动

1. 打开 IDEA，导入项目
2. 配置环境变量（Run -> Edit Configurations -> Environment Variables）
3. 运行主类：`com.example.ragbilibili.RagBilibiliServerApplication`

### 方式三：打包运行

```bash
mvn clean package -DskipTests
java -jar target/rag-bilibili-server-0.0.1-SNAPSHOT.jar
```

## 验证启动

启动成功后，访问：
- 服务地址：`http://localhost:8080`
- 数据库表会自动创建（Flyway 迁移）

查看日志确认：
```
Started RagBilibiliServerApplication in X.XXX seconds
```

## 常见问题

### 1. 数据库连接失败
- 检查 MySQL 是否启动
- 确认用户名密码是否正确
- 确认端口 3306 是否开放

### 2. OpenAI API 调用失败
- 检查 `OPENAI_API_KEY` 和 `OPENAI_BASE_URL` 是否正确
- 确认 API 服务可访问
- 查看日志中的详细错误信息

### 3. DashVector 连接失败
- 确认 Collection 已创建
- 检查 API Key 和 Endpoint 是否正确
- 确认向量维度是否匹配（1024）

### 4. DashScope 调用失败
- 检查 API Key 是否有效
- 确认账户余额充足
- 查看百练控制台的调用日志

## 配置总结

| 配置项 | 配置位置 | 说明 |
|--------|----------|------|
| 数据库账号密码 | `application.yml` | 直接修改 username/password |
| OpenAI API | 环境变量 | `OPENAI_API_KEY`, `OPENAI_BASE_URL` |
| DashVector | 环境变量 | `DASHVECTOR_API_KEY`, `DASHVECTOR_ENDPOINT` |
| DashScope | 环境变量 | `DASHSCOPE_API_KEY` |
| Collection 名称 | `application.yml` | 默认 `bilibili` |
| 向量模型 | `application.yml` | 默认 `text-embedding-v4` |
