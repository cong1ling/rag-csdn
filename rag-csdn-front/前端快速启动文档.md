# 前端快速启动文档

## 1. 文档目标

本文档用于启动 `rag-csdn-front` 前端项目，适合本地开发、接口联调和构建验收。

当前前端已经切换到 **CSDN 文章知识库** 语义：

- 主业务页面：`/articles`
- 兼容旧路由：`/videos`
- 默认后端接口前缀：`/api`

## 2. 技术栈

- Vue 3：构建页面与组件
- Vite：本地开发服务器与打包工具
- Vue Router：管理页面路由与鉴权跳转
- Pinia：管理用户状态与登录态
- Element Plus：提供界面组件
- Axios：统一处理接口请求与错误
- Marked + DOMPurify：渲染并清洗聊天返回的 Markdown 内容

## 3. 环境要求

- Node.js 18+
- npm 9+

查看版本：

```bash
node -v
npm -v
```

## 4. 安装依赖

```powershell
cd rag-csdn-front
npm install
```

## 5. 配置环境变量

前端环境变量模板：

```text
rag-csdn-front/.env.example
```

复制模板：

```powershell
Copy-Item .env.example .env
```

如使用 Bash：

```bash
cp .env.example .env
```

默认配置：

```env
VITE_API_BASE_URL=/api
VITE_PROXY_TARGET=http://localhost:8080
```

参数说明：

- `VITE_API_BASE_URL`：浏览器中请求接口的基础路径
- `VITE_PROXY_TARGET`：Vite 开发环境代理转发的后端地址

## 6. 启动开发环境

确保后端已启动后，再执行：

```powershell
cd rag-csdn-front
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

当前开发服务器配置：

- 监听地址：`0.0.0.0`
- 端口：`5173`
- 代理前缀：`/api`

## 7. 当前联调方式

默认联调链路如下：

1. 浏览器访问 `http://localhost:5173`
2. Vite 将 `/api/*` 请求代理到 `http://localhost:8080`
3. 登录成功后，前端把 JWT 保存到 `localStorage`
4. Axios 请求会自动附加 `Authorization: Bearer <token>`
5. 聊天页面通过 `fetch + SSE` 调用 `/api/sessions/{sessionId}/messages/stream`

说明：

- 当前前端 **不是 Cookie 会话模式**
- 当前登录态核心依赖是 JWT Bearer Token

如果后端地址不是 `8080`，只需要修改：

```env
VITE_PROXY_TARGET=http://你的后端地址
```

## 8. 主要页面

- `/`：首页
- `/login`：登录
- `/register`：注册
- `/import`：导入文章
- `/articles`：文章列表
- `/sessions`：会话列表
- `/chat/:sessionId`：流式问答页

未登录访问业务页面时，会自动跳转到登录页。

## 10. 常见问题

### 10.1 页面能打开，但接口全部报错

优先检查：

- 后端是否已经启动
- `VITE_PROXY_TARGET` 是否正确
- 浏览器网络请求是否已命中 `/api/*`
- 后端 `CORS_ALLOWED_ORIGINS` 是否允许 `http://localhost:5173`

### 10.2 登录成功后仍然被跳回登录页

优先检查：

- `/api/auth/current` 是否能返回当前用户
- 浏览器 `localStorage` 中是否存在 `rag_token`
- 请求头中是否真的带上了 `Authorization: Bearer <token>`
- 后端 JWT 配置与拦截器是否正常生效

### 10.3 聊天页面没有流式输出

优先检查：

- `/api/sessions/{sessionId}/messages/stream` 是否可访问
- 浏览器控制台是否有 401、网络中断或 CORS 错误
- 后端模型服务与 DashVector 是否正常
- 返回内容是否符合 SSE 事件流格式
