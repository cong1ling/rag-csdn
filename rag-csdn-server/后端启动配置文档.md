# RAG-CSDN 后端启动配置文档

## 1. 当前状态

后端当前已经完成从 Bilibili 视频语义到 CSDN 文章语义的迁移。

当前约定如下：

- 主包名：`com.example.ragcsdn`
- 启动类：`com.example.ragcsdn.RagCsdnApplication`
- 主业务接口：`/api/articles`
- 兼容接口：`/api/videos`
- 默认数据库：`rag_csdn`
- DashVector Collection：`csdn`

Flyway 已包含历史结构前向迁移脚本，可把旧的 `video / bvid / video_id` 语义升级到当前 `article / source_id / article_id`。

## 2. 环境要求

- Java 17+
- Maven 3.6+
- MySQL 8.0+
- OpenAI 兼容聊天模型服务
- 阿里云 DashScope
- 阿里云 DashVector

## 3. 关键配置文件

- 主配置：`src/main/resources/application.yml`
- 配置示例：`src/main/resources/application.yml.example`
- 环境变量示例：`.env.example`

## 4. 必需环境变量

建议将以下变量配置到系统环境变量、IDE 运行配置，或当前启动终端中：

```env
DB_USERNAME=root
DB_PASSWORD=123456
JWT_SECRET=replace_with_a_long_random_secret
OPENAI_API_KEY=your_openai_api_key
OPENAI_BASE_URL=https://api.openai.com
DASHSCOPE_API_KEY=your_dashscope_api_key
DASHVECTOR_API_KEY=your_dashvector_api_key
DASHVECTOR_ENDPOINT=your_dashvector_endpoint
CORS_ALLOWED_ORIGINS=http://localhost:5173
```

可选变量：

```env
REGISTER_ENABLED=true
```

说明：

- `JWT_SECRET` 必须是足够长的随机字符串，否则 JWT 生成与校验会失败
- `OPENAI_BASE_URL` 支持任意 OpenAI 兼容服务地址
- `CORS_ALLOWED_ORIGINS` 本地联调时建议设置为 `http://localhost:5173`

## 5. 数据库与向量库配置

当前默认 JDBC 连接：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/rag_csdn?createDatabaseIfNotExist=true&useUnicode=true&characterEncoding=utf8&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai
```

说明：

- 默认库名为 `rag_csdn`
- 如果数据库账号有建库权限，首次启动可自动创建数据库
- 如果数据库账号没有建库权限，请先手动创建 `rag_csdn`
- Flyway 会在启动时自动执行迁移脚本

当前默认向量配置：

```yaml
spring:
  ai:
    dashscope:
      embedding:
        options:
          model: text-embedding-v4
          dimensions: 1024
    alibaba:
      dashvector:
        collection: csdn
        default-top-k: 5
        similarity-threshold: 0.35
```

创建 DashVector Collection 时建议：

- 名称：`csdn`
- 向量维度：`1024`
- 距离度量：`cosine`

## 6. 启动项目

### 6.1 PowerShell 启动

```powershell
cd rag-csdn-server
mvn spring-boot:run
```

### 6.2 IDEA 启动

运行主类：

```text
com.example.ragcsdn.RagCsdnApplication
```

### 6.3 打包运行

```powershell
cd rag-csdn-server
mvn clean package -DskipTests
java -jar target/rag-csdn-server-1.0.0.jar
```

## 7. 启动后验证

默认访问地址：

```text
http://localhost:8080
```

可选联调页：

```text
http://localhost:8080/dev.html
```

建议按下面的顺序验证：

1. 查看控制台日志，确认应用以 `RagCsdnApplication` 启动
2. 调用注册、登录接口，确认 JWT 能正常签发
3. 调用 `GET /api/auth/current`，确认 `Authorization: Bearer <token>` 可正常鉴权
4. 导入一篇 CSDN 文章，确认文章状态能从 `IMPORTING` 变为 `SUCCESS`
5. 创建会话并访问流式消息接口，确认 SSE 正常输出

## 10. 常见问题

### 10.1 后端无法启动

优先检查：

- MySQL 是否可连接
- `JWT_SECRET` 是否已配置
- `OPENAI_API_KEY`、`DASHSCOPE_API_KEY`、`DASHVECTOR_*` 是否已配置

### 10.2 登录成功，但后续接口仍然 401

优先检查：

- 前端是否携带 `Authorization: Bearer <token>`
- `JWT_SECRET` 是否在启动期间被正确读取
- `LoginInterceptor` 是否生效

### 10.3 文章导入失败

优先检查：

- URL 是否是可访问的 CSDN 文章链接
- DashScope Embedding 是否可用
- DashVector Collection `csdn` 是否存在
- 后端日志中是否记录了失败原因

### 10.4 聊天无检索结果或回答质量差

优先检查：

- 文章是否已成功导入并完成分片
- DashVector 中是否已写入向量
- 当前会话范围是否选错
- `rag.chat.*` 检索优化配置是否被误改
